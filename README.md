# ✈️ Skytrax Airline Review Analysis

> An end-to-end data science project that collects, processes, and statistically analyzes 100k+ passenger reviews from [Skytrax](https://www.airlinequality.com/) to uncover what drives airline customer satisfaction and recommendation behavior.

---

## 📌 Project Overview

This project builds a complete pipeline — from raw web scraping to statistical modelling — to analyze airline quality across 200+ carriers. Using customer review data from Skytrax, we explore patterns in service ratings, identify distinct passenger segments, and model the factors that drive whether a customer would recommend an airline.

## 🗂️ Dataset

`Airline_review.csv` is the primary dataset used for all analysis in this project. It was generated by running the web scraping pipeline (`airline_review_web_scraping.R`), which collected and structured passenger reviews from Skytrax across 200+ airlines. Each row represents one review and includes fields such as overall rating, traveller type, seat class, and 7 service sub-ratings.

---

## 🔧 Pipeline Overview

### Phase 1 — Airline Discovery
- Fetches all airline names and profile URLs from the Skytrax A–Z index using `rvest` and `httr`
- Constructs a structured tibble of 200+ airlines with their base review URLs

### Phase 2 — Review Scraping with Pagination
- For each airline, iterates through paginated review pages (100 reviews/page) using dynamic URL construction
- Extracts structured fields per review: overall rating, title, date, traveller type, seat type, and 7 service sub-ratings (seat comfort, cabin staff, food & beverages, ground service, inflight entertainment, Wi-Fi, value for money)
- Implements polite scraping with randomized delays (`Sys.sleep`) and graceful error handling via `tryCatch`

### Phase 3 — Export
- Combines all scraped reviews into a single flat CSV (`Airline_review.csv`)

## 📊 Analysis

### Data Preprocessing
- Cleaned column names to `snake_case` using `janitor`
- Parsed `date_flown` into `fly_month` and `fly_year` using `lubridate`
- Converted `recommended` to a binary numeric variable
- Created two separate analysis datasets: `eda_data` (for EDA) and `data_test` (for modelling)

### Exploratory Data Analysis (EDA)
| Question | Description |
|---|---|
| Q1 | Top airlines by review count, recommendation rate, and average rating |
| Q2 | Airlines with the best value-for-money scores |
| Q3 | Most polarized airlines by rating standard deviation |
| Q4 | Contradiction analysis — low-rated but still recommended airlines |
| Q5 | Seasonal trends in review volume and average ratings by month |
| Q6 | Industry-wide year-over-year satisfaction trend |

### Statistical Modelling

#### Logistic Regression (Recommendation Behavior)
- **Model 1 (Baseline):** Service attributes only (seat comfort, cabin staff, food & beverages, etc.)
- **Model 2 (+Demographics):** Adds traveller type and seat class
- **Model 3 (Interactions):** Interaction terms between seat comfort × seat type and value for money × traveller type
- Model comparison via ANOVA (Chi-square) and AIC

#### Dimensionality Reduction — PCA
- Applied PCA on 7 numeric service variables (standardized)
- Extracted top 2 principal components; PC1 interpreted as overall service quality

#### Customer Segmentation — K-Means Clustering (K=3)
- Clustered passengers using the top 3 principal components
- Profiled each cluster by average service scores to identify distinct passenger experience groups
- Visualized segments on PC1 vs. PC2 space

#### Quantile Regression (Overall Rating)
- Fitted quantile regression models at the 25th, 50th, and 75th percentiles
- Analyzed how the effect of each service attribute differs for dissatisfied vs. satisfied passengers

## 📈 Key Findings

- **Value for money** and **cabin staff service** are the strongest predictors of recommendation behavior across all logistic regression models
- Airlines with **bimodal rating distributions** (e.g., Akasa Air) reflect highly inconsistent passenger experiences, often split by seat class
- K-Means clustering revealed **three distinct passenger segments**: highly satisfied, neutral/mixed experience, and consistently dissatisfied
- Quantile regression showed that **inflight entertainment and Wi-Fi** matter more for already-satisfied passengers (75th percentile) than for dissatisfied ones
